# 个人待办事项页面 PRD

## 1. 产品概述

本产品是一个纯前端个人待办事项页面，用于帮助用户快速记录、查看和管理日常任务。第一版已经完成，重点是让用户在一个清晰、轻量、可本地保存的界面里管理当天或近期任务。

## 2. 目标用户

| 用户类型 | 使用场景 |
|---|---|
| 学生 | 记录作业、考试、学习计划 |
| 上班族 | 记录每日工作任务和截止日期 |
| 普通个人用户 | 管理生活、学习、工作中的待办事项 |
| 前端学习者 | 作为纯前端项目练习和展示 |

## 3. 第一版已实现功能

### 3.1 快速添加任务

- 页面提供明显的任务添加区域。
- 输入框提示用户添加新的待办事项。
- 支持点击“添加”按钮提交。
- 支持按 Enter 快速提交。
- 不允许添加空任务。
- 添加成功后清空输入框、优先级恢复为中、截止日期清空。

### 3.2 设置优先级

- 添加任务时可选择高、中、低优先级。
- 默认优先级为中。
- 不同优先级在列表中有不同边框和标签颜色。

### 3.3 设置截止日期

- 添加任务时必须选择截止日期。
- 截止日期控件的最小值是当天。
- 列表中展示截止日期。
- 未完成任务超过截止日期时展示逾期状态。

### 3.4 任务列表展示

每条任务展示：

- 完成状态复选框
- 任务标题
- 优先级标签
- 截止日期
- 创建时间
- 编辑按钮
- 删除按钮

展示规则：

- 未完成任务正常显示。
- 已完成任务降低视觉权重，标题显示删除线。
- 逾期未完成任务突出截止日期。
- 空列表展示“暂无待办事项”提示。

### 3.5 完成和取消完成

- 点击复选框可切换完成状态。
- 再次点击可恢复未完成状态。
- 状态变化后立即保存到本地。

### 3.6 删除任务

- 每条任务提供删除按钮。
- 删除后任务从列表中移除，并立即保存到本地。

### 3.7 编辑任务

- 每条任务提供编辑按钮。
- 点击后进入行内编辑状态。
- 编辑内容不能为空。
- 支持保存和取消。
- 保存后立即更新本地数据。

### 3.8 筛选任务

支持以下筛选项：

| 筛选项 | 说明 |
|---|---|
| 全部 | 显示所有任务 |
| 未完成 | 只显示未完成任务 |
| 已完成 | 只显示已完成任务 |

当前筛选项会高亮，筛选只影响展示，不删除真实数据。

### 3.9 搜索任务

- 支持根据任务标题实时搜索。
- 空搜索展示当前筛选条件下的全部任务。
- 搜索和筛选可叠加使用。

### 3.10 拖拽排序

- 支持拖拽当前可见任务调整顺序。
- 排序结果写入任务的 `order` 字段并保存到本地。
- 在筛选或搜索状态下拖拽时，隐藏任务的位置会被保留。

### 3.11 统计信息

页面顶部展示：

- 全部任务数量
- 未完成任务数量
- 已完成任务数量

任务列表标题区域会根据当前筛选展示简短摘要。

### 3.12 本地数据保存

- 使用 `localStorage` 保存任务数据。
- 页面刷新后任务数据保留。
- 添加、删除、编辑、完成状态切换、拖拽排序都会自动保存。
- 本地数据解析失败时使用空数组兜底。

## 4. 非功能性需求

### 4.1 性能

- 任务数量在 100 条以内应保持流畅。
- 避免不必要的大型依赖和复杂动画。
- 搜索、筛选和排序逻辑保持轻量。

### 4.2 安全

- 用户输入内容必须去除前后空格。
- 不允许保存空标题。
- 渲染用户输入时使用 `textContent`，避免直接拼接 HTML。
- 数据仅保存在浏览器本地，不上传服务器。

### 4.3 可用性

- 新用户无需学习即可添加、编辑、完成和删除任务。
- 添加入口明显。
- 按钮文案清楚。
- 空状态和表单错误有提示。
- 移动端可正常使用。

### 4.4 兼容性

面向现代浏览器：

- Chrome
- Edge
- Firefox
- Safari
