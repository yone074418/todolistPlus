# DEVELOP

## 1. 项目概述

本项目是一个纯前端个人待办事项页面，用于帮助用户快速记录、查看和管理日常任务。

项目主要特点：

- 快速添加任务
- 页面清晰干净
- 添加入口明显
- 支持任务完成、删除、编辑
- 支持任务优先级区分
- 支持任务筛选
- 使用 localStorage 保存数据
- 不依赖后端和数据库

项目目标是实现一个简单、实用、易维护的待办事项页面，避免功能过多影响用户关注任务本身。

## 3. 数据模型

任务数据使用数组保存，每个任务是一个对象。

### 3.1 任务对象结构

```js
{
  id: 1,
  title: "完成待办事项页面",
  completed: false,
  priority: "medium",
  createdAt: "2026-05-18 10:00:00"
}
```

### 3.2 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Number | 任务唯一编号 |
| title | String | 任务标题 |
| completed | Boolean | 是否完成 |
| priority | String | 任务优先级 |
| createdAt | String | 创建时间 |

### 3.3 优先级说明

| priority | 含义 |
|---|---|
| high | 高优先级 |
| medium | 中优先级 |
| low | 低优先级 |

## 4. 开发要求

### 4.1 添加任务

要求：

- 输入框不能为空
- 点击添加按钮可以添加任务
- 按 Enter 键可以添加任务
- 添加成功后清空输入框
- 新任务默认未完成
- 新任务默认优先级为 medium
- 添加后保存到 localStorage
- 添加后重新渲染任务列表

---

### 4.2 展示任务

每条任务需要显示：

- 完成复选框
- 任务标题
- 优先级标签
- 编辑按钮
- 删除按钮

展示要求：

- 未完成任务正常显示
- 已完成任务文字变灰并添加删除线
- 高优先级任务要突出显示
- 中优先级任务正常显示
- 低优先级任务弱化显示

---

### 4.3 完成任务

要求：

- 点击复选框切换任务完成状态
- 已完成任务显示删除线
- 再次点击可以取消完成
- 状态变化后保存到 localStorage

---

### 4.4 删除任务

要求：

- 点击删除按钮删除对应任务
- 根据任务 id 删除数据
- 删除后重新渲染页面
- 删除后更新 localStorage

---

### 4.5 编辑任务

要求：

- 点击编辑按钮后可以修改任务标题
- 修改后的内容不能为空
- 保存后更新任务数据
- 保存后更新 localStorage
- 编辑功能保持简单，不使用复杂弹窗

---

### 4.6 筛选任务

筛选项包括：

| 筛选项 | 说明 |
|---|---|
| 全部 | 显示所有任务 |
| 未完成 | 只显示未完成任务 |
| 已完成 | 只显示已完成任务 |

要求：

- 点击筛选按钮后更新列表
- 当前筛选项高亮显示
- 筛选只影响展示，不删除真实数据

---

### 4.7 本地存储

使用 localStorage 保存任务数据。

保存数据：

```js
localStorage.setItem("todoList", JSON.stringify(todoList));
```

读取数据：

```js
const todoList = JSON.parse(localStorage.getItem("todoList")) || [];
```

要求：

- 页面加载时读取本地数据
- 添加任务后保存
- 删除任务后保存
- 编辑任务后保存
- 修改完成状态后保存

## 7. 注意事项

### 7.1 页面不要太复杂

本项目重点是个人待办事项管理，不需要加入太多复杂功能。

避免：

- 过多按钮
- 过多弹窗
- 过多颜色
- 复杂动画
- 页面信息堆叠

---

### 7.2 添加入口要明显

添加任务是核心功能，输入框和添加按钮应放在页面明显位置。

要求：

- 输入框足够宽
- 添加按钮清楚可见
- 支持 Enter 快捷添加

---

### 7.3 注意用户输入安全

任务标题来自用户输入，需要避免安全问题。

要求：

- 不直接使用 innerHTML 渲染用户输入
- 推荐使用 textContent 显示任务标题
- 输入内容需要 trim 处理
- 空内容不能提交

---

### 7.4 注意本地存储

localStorage 只能保存字符串。

因此保存数组时需要使用：

```js
JSON.stringify()
```

读取数组时需要使用：

```js
JSON.parse()
```

如果没有本地数据，默认使用空数组。

---

### 7.5 优先级不要过度夸张

优先级需要有视觉区分，但不要让颜色过于杂乱。

建议：

| 优先级 | 样式 |
|---|---|
| 高 | 红色 |
| 中 | 蓝色或橙色 |
| 低 | 灰色 |
